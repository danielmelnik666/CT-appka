Architektúra

O projekte FloraGuard

Školský projekt demonštrujúci moderný multi-cloud prístup k nasadeniu AI aplikácie.

FloraGuard je webová aplikácia na detekciu inváznych druhov rastlín pomocou neurónovej siete MobileNetV2. Aplikácia je rozdelená do troch nezávislých cloudových vrstiev, čo demonštruje reálnu produkčnú architektúru.

Frontend
Azure Static Web Apps
HTML · CSS · JavaScript
HTTPS / REST
Backend API
Azure App Service
Python · Flask
⚡ Cross-cloud
AI Inferencia
Hugging Face Spaces
MobileNetV2
⚡
Kľúčový architektonický princíp: AI inferencia prebieha na Hugging Face Spaces — inom cloud providerovi ako backend (Azure). Toto oddelenie demonštruje multi-cloud stratégiu a nezávislosť komponentov.
Azure

Frontend

  • Statický hosting bez servera
  • Globálna CDN distribúcia
  • Automatický deployment z GitHub Actions
  • Bezpečný HTTPS endpoint
Azure

Backend API

  • Flask REST API v Pythone
  • Sprostredkováva komunikáciu s AI
  • CORS konfigurácia pre SPA
  • Endpoint /api/predict · /api/ping
Hugging Face

AI Služba

  • Model MobileNetV2 — binárny klasifikátor
  • Vstup: obrázok rastliny (multipart)
  • Výstup: skóre invazívnosti (0.0–1.0)
  • Iný cloud ako backend — oddelená zodpovednosť
Databáza

PostgreSQL

  • Supabase / Azure PostgreSQL
  • Ukladanie histórie detekcií
  • Metadáta: dátum, výsledok, skóre
  • Možnosť budúceho rozšírenia
